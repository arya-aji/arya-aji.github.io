# Halo, saya Aji 👋

Kalau kita ngobrol hari ini, mungkin kamu bakal langsung ngenalin saya sebagai seorang *Software Engineer*. Tapi kalau ditarik ke belakang, sebenarnya saya nggak langsung "nyemplung" di sini. Perjalanan saya justru dimulai dari tumpukan data. Dulu, hari-hari saya penuh dengan angka—mastiin semuanya valid dan rapi. Tapi jujur, ada satu hal yang bikin saya sering "geregetan": prosesnya lambat banget, serba manual, dan rasanya nggak efisien. Insight-nya sudah dekat, tapi jalannya kok muter-muter? Dari situ muncul pertanyaan simpel: **“Kenapa harus serumit ini, sih?”**

Pertanyaan itu yang akhirnya ngerubah cara main saya. Saya nggak mau cuma jadi pengolah data lagi; saya pengen bangun "kendaraan"-nya—sesuatu yang bikin kerjaan jadi lebih cepat dan simpel. Akhirnya, saya mulai belajar ngoding lewat *script* kecil buat automasi tugas receh. Ternyata efeknya luar biasa! Kerjaaan yang biasanya bikin lembur berjam-jam, tiba-tiba bisa kelar dalam hitungan detik. *It felt like magic.*

Dari situ, saya mulai berani bangun aplikasi web, bikin *dashboard* yang beneran ngebantu orang ambil keputusan, sampai nyusun sistem yang bisa ngehapus kerjaan repetitif. Sekarang saya nggak liat data sebagai hasil akhir, tapi sebagai bagian dari sistem besar yang bisa dioptimalkan. Hari ini, fokus saya adalah ngebangun *web app* dan produk SaaS yang misi utamanya satu: bikin kerjaan sehari-hari jadi jauh lebih ringan. Karena bagi saya, teknologi yang keren itu bukan yang fiturnya paling ribet, tapi yang kerasa paling gampang pas dipake.

### Filosofi kode saya:

```typescript
function solveProblem(input: any) {
  // nggak perlu ribet, yang penting solutif
  return process(input);
}
```

Perjalanan ini masih panjang, tapi saya percaya: **kalau sesuatu bisa dibuat lebih simpel, kenapa harus dibiarin ribet?**

# Hi, I'm Aji 👋

If we met today, you’d probably tag me as a *Software Engineer*. But that’s actually the second chapter of my story; it all started with a deep dive into the world of data. I used to spend my days surrounded by numbers—validating, cleaning, and organizing. But I kept hitting the same wall: the processes were slow, painfully manual, and just felt "clunky." The data was there, the insights were waiting, but the path to get there was a total maze. That’s when I started asking myself: **“Does it really have to be this hard?”**

That single question changed everything. I realized I didn't just want to interpret data; I wanted to build the tools that handled it. I wanted to create something that made things faster and more logical. So, I picked up coding, starting with tiny scripts for basic automation. The impact was immediate—tasks that used to take half a day were suddenly finished before I could even finish my coffee.

From there, I moved on to building full-scale web apps and SaaS products. My perspective shifted; I stopped seeing data as the final destination and started seeing it as a moving part in a larger system that can be optimized. Today, I build tools not just to push code, but to solve real problems. In my book, great technology isn’t the kind that looks complex—it’s the kind that feels effortless to use.

### A simple philosophy in code:

```typescript
function solveProblem(input: any) {
  // keep it simple, make it useful
  return process(input);
}
```

The journey is still unfolding, and one thing remains constant: **if it can be made simpler, it’s worth building.**
